G+ 完整的JAVA web开发框架，完整的源码

G+前端参考Hplus风格，Hplus是一个完全响应式，基于Bootstrap3.3.6最新版本开发的扁平化主题，她采用了主流的左右两栏式布局，使用了Html5+CSS3等现代技术，她提供了诸多的强大的可以重新组合的UI组件，并集成了最新的jQuery版本(v2.1.4)，当然，也集成了很多功能强大，用途广泛的jQuery插件，她可以用于所有的Web应用程序，如网站管理后台，网站会员中心，CMS，CRM，OA等等，当然，您也可以对她进行深度定制，以做出更强系统。

G+后台整合目前常用的 Spring、hibernate框架，并进行简单封装，从而使得更方便，编码更简单；同时还提供了常用的功能，例如权限控制、RBAC模块、登录登出、异常管理等等；当然，您也可以对她进行深度定制，以做出更强系统。

G+使用参考例子包括后端的开发流程，前端html页面提供很多插件和样式，同样可以直接访问和参考

    使用说明

    正确的开启GWeb方式

    1. Git clone https://github.com/assertmyself/Gweb
    2. 阅读Readme.md,对Gweb做简单了解
    3. 如不感兴趣，直接return
    4. 对框架感兴趣的话；使用mvn将源码导入到eclipse中
    5. 导入sql脚本到Mysql中
    6. 部署到tomcat中，并访问指定URL
    7. 使用admin/admin登录成功，恭喜你，构建成功
    8. 可以作为简单的web开发框架，在后续项目使用
    9. 也可以深度开发，自由发挥
    10. ……

    如果构建问题，参考readme.md，也可学习maven,构建过程很简单。


    Maven 传送门。如果传送失败，自行百度。
    编译问题，查看mvn依赖是否下载完成，自定义包是否导入
    Eclipse无法导入问题，maven eclipse:eclipse 是否执行
    IDEA无法导入问题，，参考eclipse导入问题
    关于JDK;已知在1.8环境下编译失败，最好使用 1.6和1.7的版本

    如果运行有问题，检测下sql脚本是否导入，


    java版本建议1.6以上，不建议1.8
    Mysql版本建议5.0以上
    Tomat 建议1.6 或者1.7

    其它问题，自行解决
